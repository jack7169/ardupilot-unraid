CLI Tool (ap-build)

A command-line interface that talks to the same API as the web UI. Every build submitted via CLI appears on the web dashboard.

Installation

cp ap-build /usr/local/bin/

Requires curl and jq.

Environment

VariableDefaultDescription
AP_BUILD_APIhttp://carthagenas.local:8000/api/v1API base URL

For public access: AP_BUILD_API=https://jforbes.us/api/v1

List available options

# List all vehicle types
ap-build list vehicles

# List versions for a vehicle (shows remote, type, and version ID)
ap-build list versions plane

# List boards for a specific version
ap-build list boards plane <version_id>

# List features with defaults for a board
ap-build list features plane <version_id> CubeOrange

# List recent builds (optionally filter by state)
ap-build list builds
ap-build list builds --state RUNNING

Submit a build

# Build with default features
ap-build submit plane <version_id> CubeOrange

# Build with all features enabled
ap-build submit plane <version_id> CubeOrange --all-features

# Build with no optional features
ap-build submit plane <version_id> CubeOrange --no-features

# Build with specific features
ap-build submit plane <version_id> CubeOrange --features HAL_ADSB_ENABLED,GPS

Monitor builds

# Check build status
ap-build status <build_id>

# View build logs
ap-build logs <build_id>

# Stream logs in real-time
ap-build logs <build_id> --follow

# View last 50 lines
ap-build logs <build_id> --tail 50

Download artifacts

# Download firmware archive
ap-build download <build_id>

# Download to a specific file
ap-build download <build_id> --output firmware.tar.gz

Example workflow

# 1. Find the version ID for jack7169's latest Plane build
ap-build list versions plane | grep jack7169

# 2. Check available boards
ap-build list boards plane <version_id>

# 3. Submit a build
ap-build submit plane <version_id> CubeOrangePlus

# 4. Follow the build progress
ap-build logs <build_id> --follow

# 5. Download when complete
ap-build download <build_id>

Autotest (SITL Testing)

Run ArduPilot's autotest suite against any branch, tag, or commit. Tests build SITL firmware and execute the test target.

Submit a test

# Run a test suite against a branch
ap-build test submit Plane test.PlaneTests1b --remote jack7169 --ref feature/extpos-kalman-fusion

# Test a specific tag
ap-build test submit Plane test.PlaneTests1b --remote jack7169 --ref Plane-4.5.2

# Test a specific commit
ap-build test submit Plane test.PlaneTests1b --remote origin --ref abc123def

# With custom waf flags
ap-build test submit Plane test.PlaneTests1b --remote origin --ref master \
    --waf-configure "--debug --enable-ekf3" --waf-build "-j8"

Monitor tests

# List recent test runs
ap-build test list

# Check test status
ap-build test status <test_id>

# Stream test logs in real-time
ap-build test logs <test_id> --follow

# View last 100 lines
ap-build test logs <test_id> --tail 100

# Cancel a running test
ap-build test cancel <test_id>

Git Management

Manage the autotest container's ardupilot git repository. Add remotes, fetch branches, and checkout any ref.

Remotes and refs

# List configured git remotes
ap-build git remotes

# List branches for a remote
ap-build git branches --remote jack7169

# List tags (optionally for a specific remote)
ap-build git tags
ap-build git tags --remote jack7169

# Add a new git remote
ap-build git add-remote jack7169 https://github.com/jack7169/ardupilot-jack.git

Update (fetch + checkout)

# Fetch and checkout a branch
ap-build git update --remote jack7169 --ref master

# Fetch and checkout a tag
ap-build git update --remote jack7169 --ref Plane-4.5.2

# Fetch and checkout a commit SHA
ap-build git update --remote origin --ref abc123def

# Add a new remote and checkout in one step
ap-build git update --remote-url https://github.com/user/ardupilot.git \
    --remote-name newuser --ref main

REST API

The full API is available at /api/v1/. Interactive Swagger docs are available for each service:

  • Build API Docs (/api/docs)
  • Autotest API Docs (/autotest/api/docs)
  • Admin API Docs (/admin/api/docs)

AI Agent Discovery

A machine-readable capabilities endpoint is available for AI agents and automation:

curl http://carthagenas.local:8000/api/capabilities | jq .

Returns a structured JSON document describing every available operation across all services, including parameters, types, and example requests.
